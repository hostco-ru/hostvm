# Как включить поддержку вложенной виртуализации?

### Подготовка хостов для работы вложенной виртуализации

{% hint style="warning" %}
Критическим условием для стабильной работы родительских виртуальных машин является наличие поддержки вложенной виртуализации на хост-сервере. Перенос такой машины на неподготовленный хост приведет к сбою в работе её дочерних ВМ. Чтобы избежать этого, необходимо включить эту опцию на всех хостах кластера, или же настроить политики миграции, запрещающие перемещение на узлы без соответствующей поддержки.
{% endhint %}

Для включения вложенной виртуализации на хосте необходимо выполнить следующие действия:

1. Переведите хост в режим обслуживания:

* перейдите в раздел Compute->Hosts и выберите нужный хост;
* нажмите Management->Maintenance. Появится окно с подтверждением операции;
* введите причину перевода в режим обслуживания (опционально);
* нажмите OK для подтверждения.
* После успешного завершения перевода статус хоста изменится на Maintenance.

2. Подключитесь по SSH на хост, и установите пакет с хуком для вложенной виртуализации

```
dnf install vdsm-hook-nestedvt
```

3. Перезагрузите хост.
4. После перезагрузки повторно подключитесь к хосту по SSH и убедитесь, что вложенная виртуализация включена, для этого выполните команду:

```
cat /sys/module/kvm*/parameters/nested
```

Если результат — Y или 1, вложенная виртуализация уже включена. В этом случае шаги 5-6 выполнять не требуется.

5. Если на шаге 4 получен N или 0, раскомментируйте соответствующую строку в файле /etc/modprobe.d/kvm.conf в зависимости от производителя процессора:

```
#Для Intel:
options kvm_intel nested=1
#Для AMD:
options kvm_amd nested=1
```

6. Для применения конфигурации перезагрузите хост.

### Включение вложенной виртуализации на отдельных виртуальных машинах

{% hint style="warning" %}
Эта опция позволяет ВМ использовать настройки вложенности, заданные на хосте. Она также ограничивает список хостов, на которых может запускаться ВМ, и требует ручной миграции.
{% endhint %}

Для включения вложенной виртуализации на отдельных ВМ выполните следующие шаги:

1. В Портеле администратора перейдите в Compute → Virtual Machines, выберите целевую ВМ и нажмите Edit.
2. В окне Edit Virtual Machine нажмите Show Advanced Options и перейдите на вкладку Host.
3. В разделе Start Running On выберите Specific Host(s) и отметьте хосты, подготовленные для вложенной виртуализации, на которых может запускаться машина.
4. В разделе CPU Options активируйте Pass-Through Host CPU.

### Настройка сетевой конфигурации

В случае нарушения сетевой доступности виртуальных машин, работающих внутри вложенной виртуализации, необходима корректировка параметров текущего сетевого профиля фильтрации трафика vNIC Profile.&#x20;

1. Откройте веб-интерфейс управления платформой и перейдите в меню: `Network` → `vNIC Profiles`.
2. В списке профилей найдите тот, который применяется для виртуальных машин с проблемной сетью и откройте его настройки (двойным кликом или через кнопку редактирования).
3. В окне редактирования профиля найдите параметр **Network Filter** и установите для него значение **No Network Filter**.
4. Нажмите кнопку **ОК** для применения конфигурации.
5. Для применения изменений сетевого фильтра на ВМ требуется её полностью выключить и снова включить (перезагрузки внутри ВМ недостаточно).
