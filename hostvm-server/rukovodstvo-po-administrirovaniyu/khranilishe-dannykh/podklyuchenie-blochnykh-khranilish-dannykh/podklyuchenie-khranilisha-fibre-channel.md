# Подключение хранилища Fibre Channel

#### **Порядок действий**

1. В меню интерфейса HOSTVM Manager перейдите в раздел **Хранилище (Storage)** → подраздел **Домены (Domains)**.
2. Нажмите кнопку **Новый домен (New Domain)**.
3. В открывшемся окне настройте основные параметры:
   * **Имя (Name):** Укажите уникальное имя для нового домена хранения.
   * **Дата-центр (Data Center):** Выберите из списка целевой центр данных, к которому будет прикреплен домен.
     * _Примечание:_ Оставьте значение `(none)`, если подходящий центр данных еще не создан или его необходимо прикрепить к центру данных позднее.
   * **Функция домена (Domain Function):** Выберите **Data (Данные)**.
   * **Тип хранилища (Storage Type):** Выберите **Fibre Channel**.
     * _Важно:_ Неподдерживаемые для данного центра данных типы хранилищ будут недоступны для выбора.
4. В поле **Используемый хост (Host)** укажите хост, через который будет выполняться подключение к хранилищу:
   * Если это первый домен данных в центре данных, выберите любой активный хост, имеющий доступ к хранилищу.
   * Если домен данных в центре данных уже существует, необходимо выбрать хост, выполняющий роль **SPM (Storage Pool Manager)**.
   * _Пояснение:_ Взаимодействие с доменом хранения всегда осуществляется через выбранный физический хост, а не напрямую через менеджер управления.
5. **Выбор целевых LUN:**
   * После выбора типа хранилища `Fibre Channel` в нижней части окна автоматически отобразится список доступных (неиспользуемых) LUN, обнаруженных выбранным хостом.
   * Отметьте флажками необходимые LUN, которые войдут в состав домена данных.
6. (Опционально) Разверните блок **Дополнительные параметры (Advanced Parameters)** для настройки пороговых значений:
   * **Порог предупреждения о малом объеме (Warning Low Space Indicator):** Укажите процентное значение. При его достижении система начнет логировать предупреждения.
   * **Порог блокировки (Critical Space Action Blocker):** Укажите значение в ГБ. При падении свободного места ниже этого порога система заблокирует операции, потребляющие дисковое пространство.
   * **Очистка после удаления (Wipe After Delete):** При включении данные на дисках будут затираться после удаления (применяется к вновь создаваемым дискам).
   * **Сброс после удаления (Discard After Delete):** Доступно только для блочных доменов. При включении позволяет отправлять команды TRIM/UNMAP на хранилище для возврата освободившегося места.
7. Нажмите кнопку **OK**.

#### **Результат**

Новый домен данных FCP будет создан. На время инициализации и подготовки к работе он будет находиться в статусе **Заблокирован (Locked)**. После завершения процесса подготовки домен автоматически подключится к указанному центру данных и станет доступен для использования.
