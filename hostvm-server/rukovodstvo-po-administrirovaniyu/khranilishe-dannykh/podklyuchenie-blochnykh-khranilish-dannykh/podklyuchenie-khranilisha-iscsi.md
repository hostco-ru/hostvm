# Подключение хранилища iSCSI

#### Порядок действий

1. В меню интерфейса HOSTVM Manager перейдите в раздел **Хранилище (Storage)** → подраздел **Домены (Domains)**.
2. Нажмите кнопку **Новый домен (New Domain)**.
3. В открывшемся окне настройте основные параметры:
   * **Имя (Name):** Укажите уникальное имя для нового домена хранения (например, `iSCSI_Data_Domain`).
   * **Дата-центр (Data Center):** Выберите из списка целевой центр данных, к которому будет прикреплен домен.
   * **Функция домена (Domain Function):** Выберите **Data (Данные)**.
   * **Тип хранилища (Storage Type):** Выберите **iSCSI**.
4. В поле **Используемый хост (Host)** укажите хост, через который будет выполняться подключение к хранилищу:
   * Если это первый домен данных в центре данных, выберите любой **активный** хост, имеющий сетевой доступ к iSCSI-цели.
   * Если домен данных в центре данных уже существует, необходимо выбрать хост, выполняющий роль **SPM (Storage Pool Manager)**.
   * _Важно: Взаимодействие с доменом хранения всегда осуществляется через выбранный физический хост. Убедитесь, что все хосты кластера имеют доступ к данному iSCSI-хранилищу, иначе они не смогут работать с дисками виртуальных машин._
5. **Поиск и подключение к iSCSI-целям:**
   * Если необходимая iSCSI-цель не отображается в списке, нажмите кнопку **Обнаружить цели (Discover Targets)** для настройки подключения.
   * В открывшейся панели заполните параметры:
     * **Адрес (Address):** Введите IP-адрес или полное доменное имя iSCSI-хранилища.
     * **Порт (Port):** Укажите порт подключения (по умолчанию используется `3260`).
     * **Аутентификация пользователя (User Authentication):** Если хранилище использует CHAP, установите этот флажок и введите **Имя пользователя CHAP (CHAP username)** и **Пароль CHAP (CHAP password)**.
   * Нажмите кнопку **Найти (Discover)**.
   * В списке результатов выберите одну или несколько целей.
     * Для подключения к одной цели нажмите **Войти (Login)**.
     * Для подключения ко всем найденным целям нажмите **Войти во все (Login All)**.
   * _Пояснение: После успешного входа система отобразит все LUN, связанные с данной целью. Если требуется много путей (multipath), необходимо повторить обнаружение и вход для каждого пути._
6. **Выбор целевых LUN:**
   * После добавления целей в нижней части окна отобразится список доступных LUN. Нажмите **+** (плюс) рядом с нужной целью, чтобы развернуть список.
   * Отметьте флажками необходимые LUN, которые войдут в состав домена данных.
   * _Важно: В домен данных можно включить только неиспользуемые LUN. LUN, уже задействованные вне данной среды, будут отображаться в списке, но использовать их не рекомендуется во избежание конфликтов._
7. (Опционально) Разверните блок **Дополнительные параметры (Advanced Parameters)** для настройки пороговых значений:
   * **Порог предупреждения о малом объеме (Warning Low Space Indicator):** Укажите процентное значение. При его достижении система начнет логировать предупреждения.
   * **Порог блокировки (Critical Space Action Blocker):** Укажите значение в ГБ. При падении свободного места ниже этого порога система заблокирует операции, потребляющие дисковое пространство.
   * **Очистка после удаления (Wipe After Delete):** При включении данные на дисках будут затираться нулями после удаления (применяется к вновь создаваемым дискам).
   * **Сброс после удаления (Discard After Delete):** Рекомендуется для тонкой provisioning. При включении позволяет отправлять команды TRIM/UNMAP на хранилище для возврата освободившегося места.
8. Нажмите кнопку **OK**.

#### Результат

Новый домен данных iSCSI будет создан. На время инициализации и подготовки к работе он будет находиться в статусе **Заблокирован (Locked)**. После завершения процесса подготовки домен автоматически подключится к указанному центру данных и станет доступен для использования. При необходимости объединения нескольких путей к одному LUN выполните настройку Multipath на хостах виртуализации.
