# Подключение блочных хранилищ данных

Виртуализация HOSTVM обеспечивает поддержку блочных устройств, подключаемых как по протоколу iSCSI, так и по протоколу FC (Fibre Channel). Логика работы с данными типами хранилищ идентична.

Для обоих протоколов домен хранения создается на основе **группы томов (Volume Group)**, которая предварительно формируется из выделенных LUN, доступных хосту виртуализации.

{% hint style="warning" %}
Для корректного создания домена хранения и обеспечения стабильной работы кластера необходимо соблюдение следующих обязательных требований к блочному хранилищу:

1. **Доступность на уровне блоков:** Каждый хост виртуализации, входящий в кластер, должен иметь прямой доступ к физическому устройству хранения (LUN) на уровне блоков посредством протоколов iSCSI или Fibre Channel.
2. **Уникальность ресурсов:** Созданные группы томов и входящие в их состав LUN являются уникальными ресурсами. **Запрещается** одновременно использовать одни и те же LUN или группы томов в нескольких доменах хранения.
3. **Идентичность конфигурации Multipath и WWID:**
   * На всех хостах кластера должна быть выполнена идентичная настройка конфигурации `multipath`.
   * Используемый LUN обязан иметь строго одинаковый WWID (WWN) на всех хостах кластера. Использование LUN с расходящимися WWID на разных узлах не допускается.
4. **Размер логического блока:** Виртуальная среда поддерживает блочные хранилища **только с размером логического блока 512 байт**. Подключение хранилищ с размером блока 4 КБ не поддерживается и может привести к ошибкам инициализации домена или некорректной работе виртуальных машин.
{% endhint %}

Если вы планируете использовать блочное хранилище и разворачивать виртуальные машины на «сырых» (raw) устройствах или напрямую подключённых LUN с дальнейшим управлением через LVM, настройте фильтр для исключения гостевых логических томов. Это необходимо, чтобы они не активировались во время загрузки хоста — в противном случае возможна потеря актуальности томов и повреждение данных. Для настройки фильтров LVM используйте команду vdsm-tool config-lvm-filter.

{% hint style="warning" %}
Перед подключением хранилища необходимо убедиться, что идентификатор WWID блочного устройства идентичен на всех узлах кластера. Несоблюдение данного требования может привести к ошибке на этапе активации, способной оказать негативное влияние на работоспособность всего кластера и управляющей машины (HOSTVM Manager).
{% endhint %}
