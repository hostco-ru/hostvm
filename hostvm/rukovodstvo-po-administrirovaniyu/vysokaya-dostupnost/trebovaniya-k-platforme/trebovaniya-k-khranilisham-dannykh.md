# Требования к хранилищам данных

Правильный выбор платформы для хранения данных позволяет избежать проблем при эксплуатации системы. Высокая производительность подсистемы хранения данных является критичной для общей производительности платформы. Дизайн инфраструктуры должен учитывать требования к резервному копированию, репликации данных, нагрузку, создаваемую приложениями. Подсистема хранения данных HOSTVM должна включать следующие элементы:

* резервирование Ethernet или FC коммутаторов, используемых для трафика подсистемы хранения данных;
* использование нескольких сетевых интерфейсов, их агрегация, при использовании iSCSI или NFS;
* настройка multipathing при использовании SAN. Резервирование HBA (FC) или инициаторов (iSCSI);
* при использовании SAN для хранилищ ВМ, рассмотрите возможность загрузки с SAN самих хостов (boot from SAN). Такая конфигурация позволит избежать проблем с локальным хранилищем хоста, а также улучшить производительность в таких задачах, как клонирование образов дисков, ускоряя время развертывания ВМ.
