# Руководство администратора

Руководство администратора предназначено для специалистов, которые:

* подключают брокеры HOSTVM VDI к HOSTVM XRM Director;
* формируют группы основной и резервной площадки;
* создают задания миграции конфигурации;
* генерируют и запускают планы восстановления;
* анализируют журналы выполнения;
* проверяют результат переноса конфигурации на резервный брокер.

***

#### Что позволяет делать HOSTVM XRM Director

HOSTVM XRM Director предоставляет единый веб-интерфейс для управления брокерами HOSTVM VDI и автоматизации операций, связанных с катастрофоустойчивостью инфраструктуры.

Система позволяет:

* регистрировать брокеры HOSTVM VDI в едином интерфейсе;
* быстро открывать веб-интерфейс конкретного брокера из HOSTVM XRM Director;
* объединять брокеры в логические группы «основная площадка / резервная площадка»;
* создавать задания миграции конфигурации;
* формировать план переноса конфигурации сервис-пулов;
* запускать перенос конфигурации на резервный брокер;
* анализировать результат выполнения по журналам операций;
* проверять факт воспроизведения конфигурации на резервной площадке.

***

#### Логика работы администратора

Типовой цикл администрирования в HOSTVM XRM Director включает следующие этапы:

1. Первичный вход и начальная проверка интерфейса.
2. Подключение брокеров HOSTVM VDI.
3. Проверка доступности основной и резервной площадки.
4. Создание группы брокеров.
5. Создание задания миграции.
6. Генерация плана восстановления.
7. Запуск миграции.
8. Анализ журналов.
9. Проверка результата на резервном брокере.

***

#### Типовой сценарий, используемый в руководстве

В рамках примера рассматривается следующая схема:

* `Broker1` — основная площадка;
* `Broker2` — резервная площадка;
* сервис-пулы для переноса: `testpool` и `test2pool`;
* группа брокеров: `Group 1`;
* задание миграции: `DailyBackupJob`.

{% hint style="info" %}
Названия брокеров, групп и заданий в рабочей среде могут отличаться. В руководстве они используются как пример административного сценария.
{% endhint %}
