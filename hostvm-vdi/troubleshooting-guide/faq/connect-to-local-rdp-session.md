# Подключение к локальной RDP сессии

В большинстве случаев реализация RDP-сервера на Linux по умолчанию настроена на создание новой сессии при каждом входящем подключении, игнорируя уже запущенный локальный рабочий стол.&#x20;

В случае, если нужно удаленно подключаться к сессии, уже запущенной локально, то необходимо произвести соответствующие настройки.

### Astra Linux

Удалённое подключение к локальной сессии возможно при использовании fly-dm-rdp. [Инструкция по настройке](https://wiki.astralinux.ru/pages/viewpage.action?pageId=321815341)

### Ubuntu

Удалённое подключение к локальной сессии возможно при использовании gnome remote desktop.

**Алгоритм настройки ОС с графической оболочкой:**

1. Установите пакет gnome-remote-desktop, если его еще нет в системе:

```
sudo apt update
```

```
sudo apt install gnome-remote-desktop
```

2. Перезагрузите машину.
3. Откройте Settings(Настройки), в разделе System -> Users, включите опцию Automatic Login для соответствующего пользователя.
4. Перейдите в System -> Remote Desktop, выберите Remote Desktop для удаленного подключения или Desktop Sharing для совместного доступа.
