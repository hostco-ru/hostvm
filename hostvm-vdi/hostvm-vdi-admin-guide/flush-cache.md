# Очистка кэша

Для очистки кэша брокера HOSTVM VDI, в `Панели управления` перейдите в раздел `Инструменты` и выберите `Очистить кэш`.

Наиболее распространёнными причинами очистки кэша являются:

* **Блокировка пользователя:** если пользователь ввёл неверный пароль максимально допустимое количество раз (задается в параметре maxLoginTries раздела Конфигурация/Security), система временно блокирует такого пользователя. Для немедленной разблокировки необходимо очистить кэш.
* **Обновление сведений:** если при редактировании настроек базового сервиса провайдера некоторые элементы, такие как хранилища данных, сети, виртуальные машины и т.п., добавленные недавно, не отображаются, необходимо очистить кэш для обновления информации (получаемые от платформы виртуализации данные кэшируются, чтобы избежать избыточных запросов). После очистки брокер вновь сделает запрос к гипервизору и обновит данные.
